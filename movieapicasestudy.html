Movie API

Case Study

Case Study

Overview

This was the first API I have ever built and it's a Movie API that provides access to information about different movies, directors, and genres. Users can sign up, update their personal information, and list their favorite movies. The API is built with Node.js and uses MongoDB for data storage.

Purpose & Context

It’s no longer enough for a JavaScript developer to be skilled in frontend development alone. It’s become essential that you be able to interface with and even create your own APIs. For this reason, I had to learn how to create a REST API for an application called “myFlix” that interacts with a database that stores data about different movies.

Objective

The objective was to learn and build the server-side component of a “movies” web application. The web application had to provide users with access to information about different movies, directors, and genres. Users had to be able to sign up, update their personal information, and create a list of their favorite movies.

Duration and credits

It took a bit more than I expected, because I needed better understanding of authentication and authorization problems and how to articulate the use of Postman to check my endpoints, which is vital in this project (and also in so many others). Here I can once more remember to give a big thanks to my previously mentioned friend, who has been with me all along, and also my mentor, who helped me a bit with Postman.

The Code

Key Features

  • User Authentication: Users can sign up and update their personal information.
  • Movie Information: Access detailed information about movies, directors, and genres.
  • Favorite Movies: Users can add movies to their list of favorites.
  • JWT Authentication for secure authentication.
  • Express.js: Built with Express for handling API requests
  • Mongoose: Used for interacting with MongoDB

    • Technologies

  • Node.js
  • Express.js
  • MongoDB (via Mongoose)
  • JWT (JSON Web Token) for authentication
  • Passport.js for handling authentication strategies
  • bcrypt for password hashing
  • dotenv for managing environment variables
  • body-parser for parsing incoming request bodies
  • cors for handling cross-origin requests
  • morgan for logging HTTP requests
  • express-validator for validating input data

    • Challenges & Solutions

    Alright, so picture me confidently opening Postman, ready to test my API like a pro. I type in the endpoint, hit send, and—access denied. No problem, just a small hiccup, right? I tweak a few things, try again… nope. My API is basically ghosting me. Turns out, JWT authentication was standing at the door, arms crossed, refusing to let me in. And Postman? Let’s just say we weren’t exactly on speaking terms. I fumbled around, trying every possible button and header configuration like a lost tourist pressing random elevator buttons. The real kicker? I had only actually figured out JWT properly in another project—after already building this API. Eventually, I cracked the code, got Postman to cooperate, and finally accessed my own endpoints. Moral of the story? Sometimes, the hardest part of building an API isn’t writing it—it’s convincing Postman to talk to it.
